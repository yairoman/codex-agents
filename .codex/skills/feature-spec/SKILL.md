# Feature Specification

## Goal

Transform a vague request into a clear implementation spec.

## Rules

- Clarify scope
- Define acceptance criteria
- Identify edge cases
- Capture assumptions
- Make lane boundaries and cross-lane dependencies explicit when relevant

## Output

1. Problem statement
2. Scope
3. Out of scope
4. Acceptance criteria
5. Edge cases
6. Scope boundaries
7. Lane inputs and outputs
8. Cross-lane dependencies
9. Assumptions
