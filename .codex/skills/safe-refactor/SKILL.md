# Safe Refactor

## Goal

Improve or modify code without breaking existing behavior.

## Rules

- Preserve public interfaces
- Reuse existing abstractions
- Keep diffs minimal
- Avoid unrelated changes

## Output

1. Files modified
2. Behavior preserved
3. Risks
4. Validation steps
